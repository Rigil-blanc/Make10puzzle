# メイクテンパズル（Make 10 Puzzle）ゲーム仕様書

本仕様書は、0〜9までのランダムな4つの数字と四則演算およびカッコを用いて10を作る「メイクテンパズル」の構成とゲーム仕様について定義します。

---

## 1. 概要
- **目的**: 与えられた4つのカード（数字）を四則演算と組み合わせ順（カッコ）によって合算・計算し、最終的に1枚のカードかつ値をぴったり「10」にすること。
- **言語**: 日本語
- **構成**: シングルビュー構成で、直感的かつ高品質なUIアニメーション、サウンドフィードバックとともにゲームを提供。

---

## 2. コア機能

### 2.1 ゲームモード
- **チャレンジモード（Challenge Mode）**
  - プレイヤーがスコアを競うタイムアタック型のモード。
  - **制限時間**: 1分、2分、3分、4分、5分から選択可能（初期設定：1分）。
  - **タイムボーナス**: 1問正解するごとに、残り時間に **+5秒** を加算（選択した制限時間の上限を越えない範囲）。
  - **コンボシステム**: 連続正解によりコンボがカウントされ、視覚効果が有効化される。
  - **ハイスコア保存**: ローカルストレージ（`localStorage`）を用いた最高記録の保持。
- **練習モード（Practice Mode）**
  - 不正解によるゲームオーバーやタイムアップがなく、自分のペースで計算を追求できる練習用モード。
  - **時間制限**: 無制限（タイマーの代わりに無限アイコンが表示される）。
  - **スコア**: カウントされない（ハイスコアに影響しない）。
  - **退出機能**: いつでもゲームを中断してタイトル（初期画面）に安全に戻ることができる。

### 2.2 難易度設定 (Difficulty)
パズルの自動生成システムにおいて、以下の4つの難易度カテゴリを設定。プレイヤーがタイトル画面で選択し、その難易度の条件に合致する問題のみが自動生成される。

| 難易度 | 定義・条件 | 例題候補 |
| :--- | :--- | :--- |
| **Easy** | 加算（`+`）と減算（`-`）の組み合わせ（足し算引き算）のみで10にできる。 | `[1, 2, 3, 4]` (1+2+3+4 = 10) |
| **Normal** | 加算（`+`）、減算（`-`）、乗算（`*`）（足し算引き算掛け算）のみで10にできる（割り算は不要）。 | `[1, 2, 3, 5]` (2*3+5-1 = 10) |
| **Hard** | 四則演算すべて（割り算を含む）とカッコを用いて10にできるが、計算過程で分母が1以外（分数）にならない（常に割り切れて整数になる）。 | `[2, 4, 8, 9]` (4/2 + 8 = 10) |
| **VeryHard** | 四則演算とカッコをフル活用し、計算過程で分数（分母が1以外）を経由することがある（経由する解法も存在する）問題。 | `[3, 3, 8, 8]` (8 / (3 - 8/3) = 10) |

## 3. 操作仕様
1. **カード選択**: 任意のカード（数字）を1枚タップして選択状態にする。
2. **演算子選択**: 画面下の「＋、－、×、÷」ボタンから1つタップして選択する。
3. **マージ処理**: もう1枚のカードをタップすると、2つのカードをマージして新たな式カードとする。
   - 例：`8` を選んで `÷` を選んだ後に `4` を選ぶと、`2`（`(8 / 4)`）のカード1枚に置き換わりマージされる。
4. **分数保持機能**: 割り切れない割り算（例: `8` ÷ `3`）の際、実数（小数）を使わず、正確に `8/3` という**分数**として値を保持・表示する。
5. **リセット & 戻す**:
   - **戻す（Undo）**: 直前のマージ処理前の配置へと1ステップ戻ることができる。
   - **リセット（Reset）**: 初期状態（4つの個別数字）に再配置する。
6. **パス**:
   - 解けない・時間がかかりすぎる場合にパスし、次の問題へ遷移可能。
7. **ヒント**:
   - 押下することで、その問題から10を作るための数式解答例（1通り。複数解がある場合は切り替え可能）をスライドイン式の解説ウィンドウに表示する。

---

## 4. サウンド・ビジュアル仕様
- **サウンド（音響合成）**: Web Audio APIを使用（外部アセット不要）。
  - カード選択、カード合体（マージ）、正解、失敗（0除算など）、パス、およびタイムリミット10秒前からのカウントダウン音を搭載。
- **UIデザイン**:
  - デスクトップ表示での整列、モバイル表示でのタッチターゲットの最適化（44px以上）。
  - 各種状態遷移（スライドイン／ポップアップなど）に `motion/react` アニマティクスを採用。
